# Domain Error File Conventions

## Simple Errors

Use `baseErr` string type declared as `const` — immutable, cannot be overwritten:

```go
type baseErr string
func (e baseErr) Error() string { return string(e) }

const (
    ErrInvalidCredentials baseErr = "invalid credentials"
    ErrAccountLocked      baseErr = "account locked"
)
```

## Errors with Context

Use structs with `Error` suffix and constructor functions:

```go
type VersionConflictError struct {
    Current  string
    Expected string
}
func (e VersionConflictError) Error() string { return fmt.Sprintf("version conflict: current %s, expected %s", e.Current, e.Expected) }
func NewVersionConflictError(current, expected string) VersionConflictError { return VersionConflictError{Current: current, Expected: expected} }
```

## Rules

- NEVER use `errors.New()` or `var Err = errors.New(...)` — use `baseErr` consts instead
- NEVER use `fmt.Errorf()` to create domain errors — use struct types with constructors
- NotFound, AlreadyExists errors are already generated by `boilerplate-go` — do not redeclare them
- Each `{module}service/errors.go` must declare `baseErr` type if it uses simple errors

## Handler Mapping

After adding a new domain error type, you MUST also add it to the `mapError` function in the corresponding `{module}connect/` handler package (e.g. `errors.go` or `handler.go`). This function maps domain errors to ConnectRPC status codes using `errors.As`:

```go
func mapError(err error) error {
    // baseErr consts — match with errors.Is:
    if errors.Is(err, authservice.ErrInvalidCredentials) {
        return connect.NewError(connect.CodePermissionDenied, err)
    }

    // Struct errors — match with errors.As:
    var versionConflict authservice.VersionConflictError
    if errors.As(err, &versionConflict) {
        return connect.NewError(connect.CodeFailedPrecondition, err)
    }

    return err // unknown errors fall through to error interceptor
}
```

Common mappings:
- Validation / bad input → `connect.CodeInvalidArgument`
- Not found → `connect.CodeNotFound`
- Already exists → `connect.CodeAlreadyExists`
- Permission / auth → `connect.CodePermissionDenied`
- Conflict / precondition → `connect.CodeFailedPrecondition`
